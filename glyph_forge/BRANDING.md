# Glyph Forge — Branding & Positioning

**North Star**: Make terminal art feel cinematic, immediate, and shareable.

## 1) Core Positioning

- **One-liner**: "Pixels to Symbols. Cinematic ASCII in one command."
- **Promise**: Instant transformation of images, video, and live streams into high‑fidelity ASCII/ANSI art.
- **Primary Value**: It looks wildly good *fast*, in a place people already are (terminal).
- **Secondary Value**: It’s programmable, scriptable, and deeply configurable.

## 2) Target Audiences

- **Creators**: Designers, digital artists, VJs, terminal aesthetics lovers.
- **Developers**: CLI/tooling fans, streamers, data artists, infra/ops.
- **Community**: r/ASCIIArt, r/commandline, terminal themers, hacker-news crowd.

## 3) Brand Voice

- **Tone**: Precise, bold, playful, slightly mythic.
- **Rules**:
  - Be *confident* in visuals, *humble* in requirements.
  - Use short, vivid verbs: "forge", "render", "transform", "stream".
  - Avoid over‑technical jargon in outward messaging.
  - Keep technical depth in docs and advanced usage.

## 4) Taglines (Pick 1-2 for consistency)

- "Pixels to Symbols. Cinematic ASCII in one command."
- "Forge beauty in the terminal."
- "Your terminal just learned cinematography."

## 5) Visual Identity (Terminal‑First)

- **Primary motif**: High‑contrast ASCII frames with a minimal border.
- **Signature asset**: `soupskull.ansi` / `soupskull_detailed.svg` as a hero preview.
- **Colors**: ANSI256 “metallic” palette; avoid neon overload.
- **Spacing**: Always show at least one framed example in README or release posts.

## 6) Product Naming System

- **Glyph Forge** (product)
- **Imagize** (image conversion)
- **Bannerize** (text to ASCII)
- **Stream** (video/YouTube/webcam)

## 7) Messaging Pillars

1) **Instant** — one command, instant output
2) **Cinematic** — it looks like art, not a demo
3) **Open‑Ended** — scriptable, composable, hackable
4) **Fast** — optimized; real‑time streaming

## 8) Copy Patterns

- **Feature**: "Braille sub‑pixel rendering"
- **Benefit**: "8x effective resolution per character"
- **Proof**: "60fps target, 1080p presets"

Example:
"Braille sub‑pixel rendering gives 8x effective resolution per character—so your terminal looks like a movie, not a blocky demo."

## 9) Viral Hooks (Product‑Level)

- One‑command demo examples in README and `--demo` mode.
- Auto‑save best render to `./glyph_forge_output/` with timestamp.
- “Copy to clipboard” output for banners (cross‑platform).
- Built‑in `--share` option to export SVG/HTML/PNG quickly.

## 10) Consistency Checklist

- [ ] README shows a cinematic example above the fold.
- [ ] CLI help text starts with a one‑line value proposition.
- [ ] Default output uses a signature frame style.
- [ ] Releases include a 5‑second demo clip or GIF.
